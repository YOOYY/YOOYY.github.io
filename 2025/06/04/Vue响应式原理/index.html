Vue2响应式原理

发表于2025-06-04|更新于2025-06-08|Vue
|浏览量:

楔子

问：假设有一个数据 data 和依赖函数 patch ， 怎样做到 data 数据改变， patch 重新执行，换句话说，怎么监听 data 数据的改变？

答：元编程， defineProperty 或 Proxy ，vue2采用的是 defineProperty ，主要是需要兼容旧版本浏览器（没错，IE，我就是在点你）

依赖收集与触发

我们需要在get中收集依赖函数 patch ，并在set中调用 patch 。
而且依赖函数的名称不一定是 patch ，可能是任何名称，但是我们收集时需要时固定的名字，所以可以把 patch 暂时放在 window.target 上，收集时只要收集 window.target 就好了

发布订阅模式

这样写有点耦合，我们按照发布订阅的模式封装一下，把依赖搜集封装成Dep类，把依赖函数封装成Watcher类。既然有了Dep类，那我们可以把window.target改成Dep.target, 毕竟window对象上的自定义属性是公共的，可以被任意修改，容易和其他库冲突

封装响应式对象

defineReactive 只能把对象中的某一个属性转为响应式，所以需要一个类遍历对象进行转化， defineReactive 也需要能递归转换子对象

数组的响应式封装

数组与对象不同，一方面数组经常使用 push 等方法改变自身的值，另一方面使用 defineProperty 处理数组也意义不大，数组的值不像对象那样可以提前定义，经常是初始化一个空数组然后向数组中 push 新值，所以数组的处理需要新的方式。

总结

Vue中响应式处理主要是采用发布订阅的模式，主要角色有 Dep ——依赖收集与发布， Watcher ——订阅者，和 Observer ——响应式转换；对于对象使用 Object.defineProperty API, 在 getter 中收集依赖，在 setter 中触发依赖；对于数组使用覆写方法的方式，在父级对象的 key getter 中收集依赖，在自定义方法中触发依赖
